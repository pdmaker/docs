# ChatGPT Next Web 集成指南

ChatGPT Next Web 是一款精心设计的 ChatGPT 网页客户端，支持一键部署和多种 AI 模型。通过 API易，您可以轻松在其中使用各种大语言模型。

## 部署方式

### Vercel 一键部署（推荐）

1. 点击 [一键部署按钮](https://vercel.com/new/clone?repository-url=https://github.com/Yidadaa/ChatGPT-Next-Web)
2. 在 Vercel 中设置环境变量：
   - `OPENAI_API_KEY`：您的 API易 密钥
   - `BASE_URL`：`https://api.apiyi.com`
3. 完成部署

### Docker 部署

```bash
docker run -d \
  --name chatgpt-next-web \
  -p 3000:3000 \
  -e OPENAI_API_KEY="您的API易密钥" \
  -e BASE_URL="https://api.apiyi.com" \
  yidadaa/chatgpt-next-web
```

### 本地开发部署

```bash
# 克隆项目
git clone https://github.com/Yidadaa/ChatGPT-Next-Web.git
cd ChatGPT-Next-Web

# 安装依赖
npm install

# 配置环境变量
cp .env.example .env.local
# 编辑 .env.local 文件，添加您的配置

# 启动开发服务器
npm run dev
```

## 配置 API易

### 基础配置

1. 打开 ChatGPT Next Web 应用
2. 点击左下角的"设置"按钮
3. 在设置页面中配置：

#### OpenAI 模型配置

- **API Key**：输入您的 API易 密钥
- **接口地址**：`https://api.apiyi.com`
- **自定义模型**：留空（使用默认模型列表）

#### 非 OpenAI 模型配置（v2.13.0+）

要使用 Claude、Gemini 等非 OpenAI 模型：

1. 在"自定义模型"输入框中添加模型
2. 格式：`+模型名称@OpenAI`
3. 示例：
   - Claude 3：`+claude-3-opus-20240229@OpenAI`
   - Gemini Pro：`+gemini-pro@OpenAI`
   - 通义千问：`+qwen-plus@OpenAI`

## 功能使用

### 对话功能

#### 新建对话
- 点击左侧"新的聊天"按钮
- 选择要使用的模型
- 开始对话

#### 模型切换
在对话过程中可以随时切换模型：
1. 点击顶部的模型名称
2. 从下拉列表选择新模型
3. 后续对话将使用新模型

#### 对话设置
每个对话可以独立设置：
- **系统提示词**：定义 AI 的角色和行为
- **温度**：控制回复的创造性（0-2）
- **历史消息数**：包含的历史对话轮数
- **回复令牌数**：控制回复长度

### 预设提示词

ChatGPT Next Web 内置了丰富的提示词模板：

1. 点击对话框上方的"/"图标
2. 浏览或搜索提示词
3. 点击使用

创建自定义提示词：
1. 进入设置 > 提示词
2. 点击"新建"
3. 编写提示词模板
4. 保存供后续使用

### 面具功能

面具功能让您可以创建预设的 AI 角色：

#### 创建面具
1. 点击左侧"面具"标签
2. 点击"新建面具"
3. 设置：
   - 面具名称
   - 面具头像
   - 系统提示词
   - 模型参数

#### 使用面具
1. 在面具列表中选择
2. 点击"使用此面具对话"
3. AI 将按照面具设定回复

### 导出功能

支持多种导出格式：
- **Markdown**：适合文档整理
- **图片**：方便分享
- **PDF**：正式文档保存

导出步骤：
1. 点击对话右上角的"..."菜单
2. 选择"导出"
3. 选择导出格式

## 高级设置

### 访问控制

设置访问密码保护您的应用：

1. 部署时设置环境变量 `CODE`
2. 多个密码用逗号分隔：`CODE=password1,password2`
3. 用户访问时需要输入密码

### 自定义域名

如果使用 Vercel 部署：
1. 在 Vercel 项目设置中
2. 添加自定义域名
3. 配置 DNS 解析

### 环境变量配置

重要的环境变量：

```bash
# API 配置
OPENAI_API_KEY=您的API易密钥
BASE_URL=https://api.apiyi.com

# 访问控制
CODE=您的访问密码

# 模型配置
DEFAULT_MODEL=gpt-3.5-turbo
CUSTOM_MODELS=+claude-3-opus-20240229@OpenAI,+gemini-pro@OpenAI

# 其他设置
HIDE_USER_API_KEY=1  # 隐藏用户 API 密钥输入
DISABLE_GPT4=0       # 是否禁用 GPT-4
```

## 常见问题

### 无法连接到 API

**问题**：显示"网络错误"或"无法连接"

**解决方案**：
1. 检查 API 地址是否正确：`https://api.apiyi.com`
2. 确认 API 密钥有效
3. 检查网络连接
4. 尝试刷新页面

### 模型不显示

**问题**：模型列表为空或缺少某些模型

**解决方案**：
1. 确保使用 v2.13.0+ 版本
2. 正确添加自定义模型
3. 检查模型名称格式
4. 清除浏览器缓存

### 回复中断

**问题**：AI 回复突然停止

**解决方案**：
1. 检查账户余额
2. 减少最大回复长度
3. 尝试其他模型
4. 检查网络稳定性

## 使用技巧

### 1. 提示词优化

使用结构化提示词获得更好效果：

```markdown
# 角色设定
你是一位经验丰富的[具体角色]

# 任务说明
请帮我[具体任务]

# 输出要求
- 要求1
- 要求2
- 要求3

# 示例（可选）
输入：xxx
输出：xxx
```

### 2. 模型选择策略

根据任务选择合适的模型：

| 任务类型 | 推荐模型 | 原因 |
|---------|---------|------|
| 日常对话 | GPT-3.5-Turbo | 快速、经济 |
| 代码编写 | GPT-4 | 准确度高 |
| 长文本分析 | Claude 3 | 上下文窗口大 |
| 创意写作 | Claude 3 | 创造力强 |
| 多语言翻译 | GPT-4 | 语言覆盖广 |

### 3. 对话管理

有效管理对话历史：

- **定期导出**：重要对话及时导出保存
- **清理历史**：删除不需要的对话释放空间
- **分类整理**：使用有意义的对话标题
- **面具分组**：不同用途创建不同面具

### 4. 性能优化

提升使用体验：

- **流式输出**：默认开启，实时看到回复
- **历史压缩**：限制历史消息数量
- **模型预热**：首次使用前先发送简单消息
- **本地缓存**：浏览器会缓存常用数据

## 安全建议

1. **保护 API 密钥**
   - 不要在公共场合展示
   - 定期更换密钥
   - 使用环境变量存储

2. **访问控制**
   - 设置访问密码
   - 限制使用人员
   - 监控使用情况

3. **数据安全**
   - 不要输入敏感信息
   - 定期清理对话历史
   - 谨慎分享对话内容
