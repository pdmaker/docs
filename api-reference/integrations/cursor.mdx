# Cursor 集成指南

Cursor 是一款 AI 驱动的代码编辑器，通过集成 API易，您可以在编写代码时获得强大的 AI 辅助功能。

## 快速配置

### 1. 打开 Cursor 设置

- 点击右上角的 **齿轮图标** ⚙️
- 在设置菜单中选择 **Models** 选项

### 2. 配置 API 密钥

1. 找到 **OpenAI API Key** 输入框
2. 输入您的 API易 密钥
   - 获取密钥：访问 [API易控制台](https://vip.apiyi.com)
   - 复制您的 API 密钥

### 3. 设置 API 地址

1. 勾选 **Override OpenAI Base URL** 选项
2. 在地址栏中输入：
   ```
   https://api.apiyi.com/v1
   ```

### 4. 验证配置

点击 **Verify** 按钮，确认配置成功。

## 模型配置

### 使用 GPT 模型

Cursor 默认支持所有 OpenAI 模型，无需额外配置即可使用：
- GPT-4
- GPT-3.5-Turbo
- GPT-4-Turbo

### 使用 Claude 模型

要在 Cursor 中使用 Claude 3.5 Sonnet，需要特殊配置：

1. 在模型列表中添加自定义模型
2. 模型名称输入：`claude-3-5-sonnet-20241022`
3. 选择并启用该模型

这样可以绕过 Cursor 的默认路由，直接使用 Claude 模型。

## 功能特性

### 代码补全

- **Tab 键补全**：输入代码时按 Tab 接受 AI 建议
- **多行补全**：支持函数级别的代码生成
- **上下文感知**：基于当前文件和项目结构提供建议

### AI 对话

- **Ctrl/Cmd + K**：打开命令面板，输入问题或指令
- **Ctrl/Cmd + L**：在侧边栏中与 AI 对话
- **代码解释**：选中代码后可请求解释或优化建议

### 代码编辑

- **生成代码**：描述需求，AI 自动生成代码
- **重构建议**：获取代码优化和重构建议
- **错误修复**：AI 协助定位和修复代码错误

## 最佳实践

### 1. 选择合适的模型

- **日常编码**：使用 GPT-3.5-Turbo，响应快速
- **复杂任务**：使用 GPT-4 或 Claude-3.5，质量更高
- **代码审查**：使用 Claude 模型，分析更细致

### 2. 优化提示词

编写清晰、具体的指令：
```
// 不好的提示
"修复这个函数"

// 好的提示
"修复这个排序函数中的边界条件错误，确保处理空数组情况"
```

### 3. 利用上下文

- 保持相关文件打开，提供更多上下文
- 在提示中引用具体的变量名和函数名
- 说明代码的预期行为和约束条件

## 常见问题

### 连接超时

如果遇到连接超时：
1. 检查网络连接
2. 确认 API 地址正确：`https://api.apiyi.com/v1`
3. 验证 API 密钥有效

### 模型不响应

如果模型无响应：
1. 检查账户余额
2. 尝试切换到其他模型
3. 重启 Cursor 应用

### 代码建议质量问题

提高代码建议质量：
1. 提供更多项目上下文
2. 使用更具体的提示词
3. 尝试不同的模型
