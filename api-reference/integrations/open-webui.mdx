# Open WebUI 集成指南

Open WebUI 是一个功能丰富的自托管 WebUI，支持各种 LLM 模型。通过 API易，您可以在 Open WebUI 中访问多种 AI 模型。

## 安装 Open WebUI

### Docker 安装（推荐）

```bash
# 使用 Docker 快速部署
docker run -d -p 3000:8080 \
  --name open-webui \
  --restart always \
  ghcr.io/open-webui/open-webui:main
```

### Docker Compose 安装

创建 `docker-compose.yml` 文件：

```yaml
version: '3.8'
services:
  open-webui:
    image: ghcr.io/open-webui/open-webui:main
    container_name: open-webui
    ports:
      - "3000:8080"
    volumes:
      - open-webui:/app/backend/data
    restart: always

volumes:
  open-webui:
```

启动服务：
```bash
docker-compose up -d
```

## 配置 API易

### 步骤 1：注册和登录

1. 访问 `http://localhost:3000`
2. 首次访问需要创建管理员账户
3. 使用创建的账户登录

### 步骤 2：进入管理设置

1. 点击右上角的个人头像
2. 选择"管理员面板"
3. 点击"外部连接"选项

### 步骤 3：配置 API 连接

在外部连接页面配置以下信息：

- **API 端点**（左侧输入框）：`https://api.apiyi.com/v1`
- **API 密钥**（右侧输入框）：输入您的 API易 密钥

点击"保存"完成配置。

## 模型管理

### 查看可用模型

配置完成后，Open WebUI 会自动获取可用模型列表。您可以在对话界面的模型选择器中看到所有可用模型。

### 常用模型

| 模型类别 | 模型名称 | 特点 |
|---------|---------|------|
| GPT 系列 | gpt-4, gpt-3.5-turbo | 通用对话，代码生成 |
| Claude 系列 | claude-3-opus, claude-3-sonnet | 长文本理解，复杂推理 |
| Gemini 系列 | gemini-pro | 多模态支持 |
| 国产模型 | qwen-plus, ernie-bot | 中文优化 |

## 使用功能

### 基础对话

1. 在主界面选择模型
2. 在输入框输入您的问题
3. 按 Enter 发送消息
4. AI 将实时流式返回响应

### 多轮对话

Open WebUI 支持上下文记忆的多轮对话：

```text
用户：请帮我写一个 Python 函数计算斐波那契数列
AI：[返回代码]
用户：能否优化这个函数的性能？
AI：[返回优化后的代码]
```

### 对话管理

- **新建对话**：点击左侧边栏的"+"按钮
- **对话历史**：左侧边栏显示所有历史对话
- **重命名对话**：右键点击对话标题
- **删除对话**：右键菜单中选择删除

## 高级功能

### 1. 系统提示词

为每个对话设置自定义系统提示词：

```text
你是一个专业的技术顾问，精通各种编程语言和框架。
请用简洁清晰的方式回答问题，必要时提供代码示例。
```

### 2. 模型参数调整

在设置中可以调整以下参数：

- **Temperature**：控制创造性（0-2）
- **Top P**：核采样参数（0-1）
- **Max Tokens**：最大输出长度
- **Presence Penalty**：重复惩罚

### 3. 文档上传

Open WebUI 支持上传文档进行分析：

1. 点击输入框旁的附件图标
2. 选择要上传的文件（支持 PDF、TXT、MD 等）
3. AI 将基于文档内容回答问题

### 4. 搜索功能

使用搜索功能快速找到历史对话：

- 按关键词搜索
- 按日期筛选
- 按使用的模型筛选

## 故障排除

### 连接问题

**症状**：无法连接到 API 或模型列表为空

**解决方案**：
1. 检查 API 端点是否正确：`https://api.apiyi.com/v1`
2. 验证 API 密钥是否有效
3. 确保网络连接正常
4. 检查防火墙设置

### 响应错误

**症状**：收到错误消息或异常响应

**解决方案**：
1. 检查账户余额是否充足
2. 确认选择的模型是否可用
3. 减少输入文本长度
4. 尝试其他模型

### 性能问题

**症状**：界面卡顿或响应缓慢

**解决方案**：
1. 清理浏览器缓存
2. 减少同时打开的对话数量
3. 使用更快的模型（如 GPT-3.5-Turbo）
4. 检查服务器资源使用情况

## 最佳实践

### 1. 模型选择建议

- **日常问答**：GPT-3.5-Turbo（快速且经济）
- **代码开发**：GPT-4 或 Claude 3（更准确）
- **创意写作**：Claude 3 或 GPT-4（更有创造力）
- **数据分析**：GPT-4（逻辑能力强）

### 2. 提示词技巧

编写有效的提示词：

```text
# 明确具体
不好：帮我写代码
更好：用 Python 写一个函数，输入列表，返回去重后的结果

# 提供上下文
不好：解释这个
更好：解释这段 React 代码中 useEffect 的作用：[代码]

# 指定格式
请用以下格式回答：
1. 概念解释
2. 代码示例
3. 注意事项
```

### 3. 安全建议

- 定期更新 Open WebUI 到最新版本
- 使用强密码保护管理员账户
- 不要在对话中分享敏感信息
- 定期备份重要对话

## 集成扩展

### API 集成

Open WebUI 提供 API 接口，可以集成到其他应用：

```python
import requests

# API 调用示例
url = "http://localhost:3000/api/chat"
headers = {
    "Authorization": "Bearer YOUR_TOKEN",
    "Content-Type": "application/json"
}

data = {
    "model": "gpt-3.5-turbo",
    "messages": [
        {"role": "user", "content": "Hello!"}
    ]
}

response = requests.post(url, headers=headers, json=data)
print(response.json())
```
