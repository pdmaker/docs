# API参考文档

欢迎来到API易的API参考文档！这里提供了完整的接口文档、调用示例和最佳实践指南。

## 🚀 API概览

API易基于统一的**OpenAI API标准**构建，完全兼容OpenAI官方API格式。这意味着：

- **无缝迁移**：现有OpenAI代码无需修改即可使用
- **统一格式**：200+模型使用相同的调用格式
- **丰富生态**：支持所有基于OpenAI API的第三方库和工具

## 📋 基础信息

### 服务地址
- **主服务地址**：`https://api.apiyi.com`
- **备用地址**：`https://vip.apiyi.com`

### 认证方式
```bash
Authorization: Bearer YOUR_API_KEY
```

### Content-Type
```bash
Content-Type: application/json
```

## 🔑 快速开始

### 1. 获取API Key
前往 [API易官网](https://api.apiyi.com) 注册并在控制台创建API Key

### 2. 第一个API调用
```bash
curl https://api.apiyi.com/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "gpt-4o-mini",
    "messages": [{"role": "user", "content": "Hello!"}],
    "max_tokens": 100
  }'
```

## 📊 HTTP状态码

| 状态码 | 含义 | 说明 |
|--------|------|------|
| 200 | 成功 | 请求成功处理 |
| 400 | 请求错误 | 参数错误或格式不正确 |
| 401 | 认证失败 | API Key无效或过期 |
| 403 | 权限不足 | API Key权限不足 |
| 429 | 请求过多 | 触发频率限制 |
| 500 | 服务器错误 | 服务器内部错误 |

## 🚨 错误处理

### 错误响应格式
```json
{
  "error": {
    "message": "错误描述",
    "type": "invalid_request_error",
    "param": "参数名",
    "code": "错误代码"
  }
}
```

### 常见错误
<AccordionGroup>
  <Accordion icon="exclamation-triangle" title="认证错误">
    ```json
    {
      "error": {
        "message": "Invalid API key provided",
        "type": "invalid_request_error",
        "code": "invalid_api_key"
      }
    }
    ```
    **解决方案**：检查API Key是否正确，确保格式为`sk-xxx`
  </Accordion>
  <Accordion icon="exclamation-triangle" title="余额不足">
    ```json
    {
      "error": {
        "message": "You exceeded your current quota",
        "type": "insufficient_quota",
        "code": "quota_exceeded"
      }
    }
    ```
    **解决方案**：前往控制台充值账户余额
  </Accordion>
  <Accordion icon="exclamation-triangle" title="模型不存在">
    ```json
    {
      "error": {
        "message": "The model does not exist",
        "type": "invalid_request_error",
        "param": "model",
        "code": "model_not_found"
      }
    }
    ```
    **解决方案**：检查模型名称是否正确，参考[模型列表](/models)
  </Accordion>
</AccordionGroup>

## 🔒 安全最佳实践

<Warning>
  **重要提醒**：请务必遵循以下安全原则，确保API Key和数据安全
</Warning>

### API Key安全
- ✅ 使用环境变量存储API Key
- ✅ 定期轮换API Key
- ✅ 为不同项目使用不同的API Key
- ❌ 不要在客户端代码中硬编码API Key
- ❌ 不要将API Key提交到版本控制系统

### 示例：环境变量配置
```bash
# .env文件
APIYI_API_KEY=sk-your-api-key-here
APIYI_BASE_URL=https://api.apiyi.com/v1
```

```python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.getenv("APIYI_API_KEY"),
    base_url=os.getenv("APIYI_BASE_URL")
)
```
